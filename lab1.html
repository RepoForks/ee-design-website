Backend Setup (Sept. 9th)

We set up a backend that uses REST-like endpoints to access our thermometer data. This way, the data itself is the service we are creating and we just have to create simple ways to access that data. The endpoints are as follows:

Add a value: http://173.17.168.19:8083/lab1/temperature/add?temp=13.65
Read previous second: http://173.17.168.19:8083/lab1/temperature/latest
Read last 300 seconds: http://173.17.168.19:8083/lab1/temperature

All responses are returned through JSON objects. The code for the backend can be found here

Web Client Setup (Sept. 9th)

After creating the client to fill the data, we wanted to be able to view those temperatures. We set up a basic webpage to just display the current temperature and get new values for it every one second.

Our web client is simple HTML, CSS, and Javascript. We used Google's Material Design Lite for responsive design, similar to this class website. The page gets rendered, then, then Javascript fills the current temperature field.

Note that this page will display 'Data Unavailable' if the backend says there is no temperature reading. So, unless the client is running and filling the values, or our temperature sensor is actually turned on, this web client will not do anything.
